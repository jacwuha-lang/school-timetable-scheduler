# 初中排课系统

一个功能完整的初中排课管理系统，支持自动排课、智能验证、冲突检测和调整建议。

## 功能特性

### 核心功能
- **自动排课**：基于约束满足算法的智能排课系统
- **多维度课表视图**：支持班级、教师、场地三种视图
- **实时冲突检测**：硬性冲突和软约束冲突实时检测
- **智能调整建议**：手调后自动生成优化建议
- **课表体检报告**：全面的课表合规性诊断

### 数据管理
- **班级管理**：管理班级信息、学生人数、班主任
- **教师管理**：管理教师信息、教授科目、工作量限制
- **科目管理**：管理科目信息、课时要求、场地需求
- **场地管理**：管理场地信息、类型、容量

### 验证与优化
- **全维度智能验证**：9种验证规则，支持自定义
- **冲突可视化**：颜色标记、图标提示、详情悬浮窗
- **智能调整建议**：3套以上可落地方案，标注推荐指数
- **统计分析**：教师工作量、科目合规性、场地使用率、班级均衡性

## 使用指南

### 1. 基础数据配置

#### 班级管理
1. 点击左侧菜单「班级管理」
2. 点击「添加班级」按钮
3. 填写班级名称、年级、学生人数
4. 保存后可继续编辑或删除

#### 教师管理
1. 点击左侧菜单「教师管理」
2. 点击「添加教师」按钮
3. 填写教师姓名、选择教授科目、设置工作量限制
4. 保存后可继续编辑或删除

#### 科目管理
1. 点击左侧菜单「科目管理」
2. 点击「添加科目」按钮
3. 填写科目名称、周课时、最小/最大周课时
4. 根据需要勾选「需要实验室」或「需要专用教室」
5. 保存后可继续编辑或删除

#### 场地管理
1. 点击左侧菜单「场地管理」
2. 点击「添加场地」按钮
3. 填写场地名称、选择场地类型、设置容量
4. 保存后可继续编辑或删除

### 2. 生成课表

1. 点击左侧菜单「课表管理」
2. 点击「生成新课表」按钮
3. 系统自动根据基础数据生成课表
4. 生成完成后可查看班级、教师、场地三种视图

### 3. 课表验证

1. 在课表管理页面点击「验证课表」按钮
2. 系统自动检测所有冲突
3. 硬性冲突显示为红色，软约束冲突显示为黄色
4. 点击有冲突的课程可查看详情和调整建议

### 4. 手动调整

1. 点击课表中的任意课程
2. 查看智能调整建议
3. 选择合适的建议并点击「执行此建议」
4. 系统自动完成调整并重新验证

### 5. 课表体检

1. 点击左侧菜单「课表体检」
2. 选择要体检的课表
3. 点击「生成体检报告」按钮
4. 查看总体评分、冲突详情、统计分析

## 验证规则说明

### 硬性冲突（一票否决）
1. **教师时间冲突**：同一教师在同一时段不能有多个教学任务
2. **班级时间冲突**：同一班级在同一时段不能有多门课程
3. **场地容量冲突**：同一场地在同一时段的占用不能超过容量上限
4. **课时定额合规**：每个班级每个科目的周课时必须在规定范围内
5. **锁定时段占用**：不能占用已锁定的时段

### 软约束冲突（优化建议）
1. **科目时段偏好**：主科应优先排在上午，体育课应避免特定时段
2. **教师教学节奏**：教师单日不应连续上课过多
3. **工作量均衡**：教师周工作量应在合理范围内
4. **班级课程均衡**：班级单日主科数量应均衡

## 数据存储

系统使用SQLite作为本地数据库，数据文件位于：
- `backend/data/schedule.db`

数据库包含以下表：
- classes（班级）
- teachers（教师）
- subjects（科目）
- rooms（场地）
- schedules（课表）
- schedule_items（课程安排）
- rules（规则）
- operation_logs（操作日志）
- health_reports（体检报告）

## 常见问题

### Q: 数据库文件在哪里？
A: 数据库文件位于 `backend/data/schedule.db`，首次运行会自动创建。

### Q: 如何备份数据？
A: 直接复制 `backend/data/schedule.db` 文件即可备份。

### Q: 如何重置系统？
A: 删除 `backend/data/schedule.db` 文件，重启后端服务即可。

### Q: 支持多学期吗？
A: 支持，生成课表时可以指定学期名称。

### Q: 可以导出课表吗？
A: 当前版本支持在课表体检报告中查看统计信息，导出功能将在后续版本中添加。
